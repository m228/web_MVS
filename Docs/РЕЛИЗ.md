# Как выпустить новую версию web_MVS

Памятка разработчику: что делать при выпуске релиза. Подробности про саму упаковку —
в [`УПАКОВКА.md`](УПАКОВКА.md).

## Перед первым релизом (один раз на машине-сборщике)

Нужна машина с **Python 3.11** и установленными зависимостями (в т.ч. камеры:
`harvesters`, `genicam`, `numpy`, `opencv`). Обычно это машина Server.

```bat
py -3.11 -m venv .venv
.venv\Scripts\pip install -r requirements.txt
```

Для публикации релизов нужен **GitHub CLI** (`gh`), авторизованный один раз:

```bat
winget install GitHub.cli      :: если gh не установлен
gh auth login
```

Репозиторий должен быть **публичным** — иначе `update.bat` на «чистых» машинах не
скачает релиз без токена.

## Выпуск новой версии — по шагам

### 1. Внести и проверить изменения в коде
```bat
py -3.11 -m py_compile camera_core.py app.py net_tools.py paths.py run.py
```

### 2. Поднять номер версии
Отредактировать файл **`VERSION`** (одна строка), напр. `1.0.0` → `1.0.1`.

Схема версий — [SemVer](https://semver.org/lang/ru/): `МАЖОР.МИНОР.ПАТЧ`
- **ПАТЧ** (1.0.0 → 1.0.1) — багфиксы;
- **МИНОР** (1.0.1 → 1.1.0) — новые функции без поломки совместимости;
- **МАЖОР** (1.1.0 → 2.0.0) — несовместимые изменения.

### 3. Закоммитить
```bat
git add -A
git commit -m "release: v1.0.1"
git push
```
(Версионный тег создаст `gh release create` на шаге 5 — отдельно `git tag` не нужен.)

### 4. Собрать бандл
```bat
build.bat
```
Соберёт `dist\web_MVS\`, добавит `run.bat` и упакует `dist\web_MVS_v1.0.1.zip`.

> Локально проверить бандл (без камеры): запустить `dist\web_MVS\web_MVS.exe`, открыть
> `http://localhost:8000` и `http://localhost:8000/api/debug/info` — должна быть новая
> `version`, `cti_source: bundled`, поднялись genicam/harvesters.

### 5. Опубликовать релиз
```bat
gh release create v1.0.1 dist\web_MVS_v1.0.1.zip -t v1.0.1 --generate-notes
```

> Тег `vX.Y.Z` должен совпадать с `VERSION`. Имя файла обязано начинаться с
> `web_MVS_` — по этой маске его ищет `update.ps1`.

### 6. Обновить копии на машинах
- запустить на машине `update.bat`, **или**
- скачать новый zip со страницы Releases и распаковать поверх старой папки.

Данные (`dataset\`, `Videos\`, `rtsp_cameras.json`) при обновлении сохраняются.
Проверить версию: `http://localhost:8000/api/debug/info`.

## Если что-то пошло не так

| Симптом | Причина / что делать |
|---|---|
| `build.bat`: `No .venv found` | нет окружения — см. «Перед первым релизом» |
| Сборка падает на `collect_all` genicam/harvesters | зависимости камеры не установлены в `.venv` |
| `update.bat`: ОШИБКА, 404 | релиз не опубликован, или репозиторий приватный |
| exe стартует, но `cti_source` ≠ `bundled` | в бандл не попала `Driver\` — проверь `web_MVS.spec` |
| Камера не открывается на целевой машине | проверять на Server: `diag.py` + панель «История событий» |

## Откат на предыдущую версию

Скачать zip нужной версии со страницы Releases и распаковать поверх. Либо пометить
старый релиз как latest:
```bat
gh release edit v1.0.0 --latest
```
