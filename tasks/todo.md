## 🔴 Критично

- [x] **#N1 Гонка `force_close` vs `generate` (GigE)** — `camera_core.py:932` vs `1044` ✅ 2b42837
  `generate()` пишет `self.ia = ia` и `self.running = True` без блокировки; `force_close()`
  читает/обнуляет `self.ia` тоже без лока. uvicorn обслуживает `/stream` и `/force_close`
  в разных потоках → поток B может уничтожить acquirer между `self.ia = ia` и `ia.start()`,
  и `start()` уйдёт на мёртвый acquirer. Фикс: короткий lock на секцию
  `[self.ia=…; self.running=True; ia.start()]` либо проверка `self.running` перед стартом.

- [x] **#N2 Утечка локального `capture` в `RtspCameraWorker.generate`** — `camera_core.py:1363` ✅ 2b42837
  При параллельном `force_close`, обнулившем `self.capture`, ветка `self.capture is not None`
  ложна → `release()` локального `capture` не вызывается → утечка сокета + потока декодирования
  FFmpeg. Фикс: независимый `if capture is not None: capture.release()` в `finally`, как для `ia`.

- [x] **#N3 Мутация `worker.interface_id` в обход `select_interface`** — `app.py:147,162,363` ✅ 49f803e
  `/api/ip`, `/api/get_network_settings`, `/api/camera/info` пишут `worker.interface_id = …`
  напрямую. Фронт долбит их параллельно → один меняет интерфейс, второй читает изменённое
  состояние и открывает камеру не через тот интерфейс. Фикс: передавать interface/handle
  параметром в `get_ip()/get_network_settings()/get_info()`, не мутируя общее состояние.

- [x] **#N4 Потеря backup при зависании apply-скрипта** — `updater.py:277` ✅ dd7763f
  `Remove-Item $backup` и `$staging` — безусловные. Если PowerShell-хелпер убьют/зависнет
  после удаления backup, но до старта нового exe — приложение мертво, восстановить нечем.
