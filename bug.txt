1. bug: Камера (GigE Vision) перестала подключаться после рефактора — "раньше работало".

   Симптом (в "История событий"):
     source: camera_core.get_node_map_cam — "Ошибка подключения к камере"
     error:  GenTL exception: Requested operation not implemented.
             (Message from the source: <мусор>) (ID: -1003)
     code:   -1003
     Изначально приходило как UnicodeDecodeError (genicam не мог декодировать
     не-UTF-8 текст ошибки продюсера) — это уже починено в _explain_error.

   Где падает:
     generate()/get_ip()/... -> open_node_map() -> manager.create_acquirer()
     -> harvester.create(). На реальной DA5896461 (MV-CS050-10GC).
     DA123123 — мок (хардкод), в genicam не идёт.

   Диагностика (diag.py на машине Server):
     - python diag.py -> STAGE 1: свежий Harvester, h.create(0) и node_map -> OK
       (Width=2448, Height=2048). Значит genicam/runtime/.cti/камера ИСПРАВНЫ.
     - Версии: python 3.11.9, genicam 1.5.1, harvesters 0.post.dev63, numpy 2.4.3.
     => Раннее предположение про обновление библиотек / нехватку runtime — НЕВЕРНО.

   ПРИЧИНА (регрессия рефактора):
     create_acquirer() перед каждым harvester.create() вызывает harvester.update()
     на ДОЛГОЖИВУЩЕМ (персистентном) Harvester. Лишний update() прямо перед create()
     дестабилизирует Hikrobot-продюсер -> create() возвращает -1003 и "мусор из
     памяти". В diag свежий Harvester делает update() ОДИН раз -> работает.
     Рабочая (до-рефактора) версия camera_core.py (коммит 8de8234) открывала так:
         ia = H.create({'serial_number': serial})   # БЕЗ update() перед create
     Рефактор добавил update() в начало create_acquirer — отсюда баг.
     (Не зря в истории есть коммит 2c44f9b "fix h.update".)

   ФИКС:
     В create_acquirer переобновляем список ТОЛЬКО если камеры ещё нет в
     device_info_list (горячее подключение). Если камера уже известна после scan
     (обычный сценарий "Подключить") — update() пропускаем, идём сразу в create().

   Доп. улучшения (диагностика):
     - _decode_gentl_message()/_explain_error(): читаемый текст + decode_error + lib_hint.
     - log_environment(): версии Python/genicam/harvesters/numpy + путь/источник .cti.
     - _discover_cti(): MVS_CTI_PATH -> GENICAM_GENTL64_PATH -> MVS SDK -> Driver/.
     - diag.py: STAGE 1 (база) / STAGE 2 (воспроизведение бага: 2-й update перед
       create) / STAGE 3 (реальный путь manager.open_node_map — с фиксом).

   Как проверить:
     - python diag.py: STAGE 2 должен дать -1003 (старое поведение), STAGE 3 — OK.
     - В приложении нажать "Подключить" на DA5896461 -> поток должен пойти.
